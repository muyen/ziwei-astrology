# 紫微鬥數資料分析框架

## metadata
- topic: 原始語料分析方法
- purpose: 從 raw/compare 資料中找出規律、衝突來源、與可解釋結構
- format: framework

## 核心問題

我們不是隻問「書上怎麼說」，而是同時問五個問題：

1. 這條規則在說什麼結構關係。
2. 這條規則和哪些其他規則重複出現。
3. 這條規則是否能被五行、宮位派生、星曜職司解釋。
4. 這條規則屬於描述性經驗，還是可推導規則。
5. 衝突版本之間，差異是抄傳、流派修正、還是底層理論不同。

## 分析層次

### 1. 詞彙層

觀察重複詞：

- 富貴
- 孤
- 貴人
- 口舌
- 暗
- 福
- 刑
- 破
- 驛馬
- 清白

目的：

- 找出每顆星、每個宮、每個格局的高頻語義核。
- 建立「古文用詞 -> 現代語義標籤」對照表。

### 2. 結構層

觀察規則結構：

- `星曜 -> 性質`
- `星曜 + 宮位 -> 應事`
- `星曜 + 四化 -> 強弱/轉向`
- `三方四正 -> 結果穩定性`
- `格局 -> 社會結果`
- `案例 -> 結構組合的實證敘述`

目的：

- 找出哪些句子只是靜態描述，哪些句子在講動態轉換。
- 找出紫微鬥數最常用的判斷單位。

### 3. 理論層

優先測試四種解釋器：

- 五行生剋
- 星曜職司
- 宮位派生
- 廟旺陷弱

每條規則都問：

- 這條能否由五行解釋？
- 這條能否由該星「主什麼」解釋？
- 這條能否由該宮的派生關係解釋？
- 這條是否高度依賴旺陷與四化？

### 4. 統計層

等 raw corpus 再多一些後，對 records 做頻率統計：

- 哪些星最常與「富貴」共現
- 哪些星最常與「孤、刑、忌、破」共現
- 哪些宮最常被定義為他宮的派生宮
- 哪些四化組合最常導向正面或負面敘事

目的：

- 區分偶發句與核心句。
- 幫助裁決流派差異時的先驗權重。

### 5. 衝突層

對每個 `conflict_group` 做單獨裁決，不混在一起。

每組衝突都做：

```yaml
- conflict_group: sihua.geng
  versions: [全書, 中州, 北派]
  compare_axes:
    - 五行是否順
    - 星曜職司是否合理
    - 是否破壞其他乾的整體對稱
    - 是否與案例庫較一致
    - 是否可能為傳抄誤字
```

## 為什麼這套書會這樣寫

先採用四個工作假說，不當成結論：

- 假說 A：大量規則是從星曜職司往外推演，再由案例反覆修正。
- 假說 B：很多宮位斷語不是孤立的，而是派生宮位系統的結果。
- 假說 C：四化衝突的核心不是記錯而已，而是不同流派對「因果流向」定義不同。
- 假說 D：格局論是古人把高維度盤面壓縮成少量可記憶模式的 mnemonic system。

## 每條規則的分析模板

```yaml
- record_id: quanshu.v1.fawei.004
  semantic_tags: [三方四正, 結構核心]
  explanation_candidates:
    - 五行: weak
    - 星曜職司: weak
    - 宮位派生: strong
    - 旺陷四化: medium
  pattern_family: 結構判盤
  empirical_or_deductive: deductive
  linked_records: [quanshu.v2.caibo.005]
  open_questions:
    - 四正定義在不同書是否完全一致
```

## 近期研究順序

1. 先把卷二八宮補齊，建立足夠的宮位派生語料。
2. 再補卷四案例，讓規則能對應真實應驗敘事。
3. 接著優先分析四化衝突，因為這是最容易做結構化裁決的一塊。
4. 最後才蒸餾成 mapping，避免過早定版。
