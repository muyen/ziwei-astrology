# 紫微鬥數技能專案總覽

## 一、我現在到底在做什麼

我現在做的不是單純「整理古籍」。

我在做的是把紫微鬥數資料拆成四層，最後做成一個真正可執行的 skill：

1. `raw`
   - 收原始古文、口訣、星曜問答、案例。
   - 保留繁體中文原文。

2. `analysis`
   - 比對衝突。
   - 找規律。
   - 找哪些規則是真正反覆出現、哪些只是後世說法。

3. `distilled`
   - 把分析壓成 runtime 可直接查的 JSON。
   - 例如四化政策、格局 family、宮位 mapping、術語正規化、機制分支、星曜向量。

4. `validation`
   - 驗證 distilled 資料是不是還能回到原文。
   - 驗證 runtime 判斷和案例是否一致。

換句話說：

我不是在做「筆記」，
我是在做「可追溯的紫微知識系統」。

## 二、最終目標是什麼

最終目標是做出一個高品質的 `紫微鬥數` skill，而且這個 skill 要同時滿足四件事：

1. 能研究
   - 能追原文
   - 能說明流派衝突
   - 能指出哪些地方仍未裁決

2. 能解盤
   - 能在 runtime 很快查到宮位、四化、格局、負向機制
   - 能給出有根據的判讀

3. 能回溯
   - 不只是給結論
   - 還能指出結論主要來自哪些 `source ids`

4. 能維護
   - 新 raw 進來後，mapping 可以繼續更新
   - 不會因為後面加資料就整套崩掉

簡化說，最終產品不是：

- 一篇很長的紫微說明文

而是：

- 一個有原文依據、有衝突管理、有 runtime 結構的紫微 skill

## 三、到目前為止有找到有用的東西嗎

有，而且不只一點。

目前最有價值的發現有三條。

### 1. `neg.limit_collision` 是目前最穩的核心機制

這是現在最有用的分析結果。

意思不是：

- 某個格局一定富貴

而是：

- 本命高格可以成立
- 但如果 `大限 + 小限 + 太歲` 和強凶信號重疊，
- 終局仍然會翻盤

這條已經被很多案例支持，
而且現在已經被壓成 runtime 分支：

- `fatal`
- `nonfatal`
- `delayed_success`
- `structural_deficit`

這是目前 skill 最有機會直接用起來的東西。

### 2. `neg.break_wealth_flow` 的原文鏈已經變強

這條一開始只有概念。

現在已經補到更直接的 raw：

- `祿馬最喜交馳`
- `忌見殺羊火截路空亡，多主勞苦`
- `馬落空亡，雖祿沖會無用主奔波`
- `與祿存同宮，謂之祿馬交馳；逢空亡謂之死馬亡馬；遇陀羅謂之折足馬`

這代表：

- `祿馬交馳` 是正向基準
- `空亡 / 截空 / 截路 / 陀羅` 這類東西會把它切斷
- 切斷之後常轉成 `奔波 / 勞苦 / 財源被劫`

所以這條機制現在在 raw 層已經很有用了。

它目前還差的是：

- 更純的案例支撐

也就是說，
不是理論不夠，
而是案例還不夠乾淨。

### 3. `截路` 現在已經不是猜詞

這也是一個很有用的進展。

之前最大問題之一是：

- `截空` 有 raw
- `截路` 比較像現代常見詞

現在已經補到古籍 raw：

- `安截路空亡訣`
- `斷橋截路大小難行`
- `天馬火，最喜會祿存，極忌截路空亡`

這表示：

- `截路` 在古籍裡是成立的
- 而且確實跟 `阻斷 / 難行 / 天馬受阻` 有關

目前剩下的問題不是「有沒有截路」，
而是：

- `截路` 和 `截空` 到底是完全同義，還是高度接近但不完全等價

這已經比之前進了一大步。

## 四、現在專案最卡的地方

現在最卡的不是 raw 量不夠。

現在最卡的是：

1. `neg.break_wealth_flow` 還缺乾淨案例
2. `star_semantic_vectors.json` 還沒有補齊
3. compare layer 還沒真正補入 `全集 / 宣微 / 十八飛星`
4. master mapping 還只是 partial，不是 final

## 五、我現在為什麼沒有繼續廣泛收案例

因為案例到 30 案之後，邊際收益下降了。

如果只是再多十個名人，
但不能解決未解機制，
那對 skill 幫助不大。

所以現在的策略是：

- 不再廣泛收案例
- 只定向收能解釋未解機制的案例

目前主要定向目標只有一條：

- `neg.break_wealth_flow`

也就是找：

- `祿馬交馳`
- `死馬 / 亡馬`
- `折足馬`
- `截路空亡`

這些更純的樣本。

## 六、我現在做的東西到底有沒有開始變成 skill

有。

現在已經不是只有分析文章，
而是有真正的 runtime 資產：

- `mapping.json`
- `runtime_policy.json`
- `mechanism_branches.json`
- `term_normalization_map.json`
- `traceability_index.json`
- `output_contract.json`
- `star_semantic_vectors.json`

而且還有實際驗證腳本：

- `scripts/validate_distilled.js`

這表示專案已經從：

- 研究資料

開始變成：

- 可執行的 skill 基礎設施

## 七、如果你現在要知道一句話總結

一句話總結就是：

這個專案現在最有用的成果，不是某一條神奇斷語，
而是已經開始把紫微鬥數拆成「可追溯、可驗證、可執行」的規則系統。
